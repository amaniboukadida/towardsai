What is Machine Learning

is a field of study of artificial intelligence, which is based on statistical approaches to give computers the ability to “learn” from data.

Machine learning is a critical technology—and it’s everywhere. You might notice an email being forwarded to spam on its own, or you might receive personalized, local recommendations from Yelp. These examples are powered by machine learning. AI is undoubtedly a key component of our future, so get ahead of the curve with Dataquest.

Machine learning is quite a large field, and in this section we list some of the most broad classes of problems in which it is interested

  1. Supervised Learning

    2. Supervised learning is perhaps the easiest type of machine learning problem to grasp: its goal is to learn how to make predictions, given a list of labeled examples, i.e. accompanied by the value to be predicted. Labels serve as a “teacher” and supervise algorithm learning..

  2. Unsupervised Learning

    3. In unsupervised learning, data is not labeled

  3. Semi-supervised learning

    4. As you might expect, semi-supervised learning involves learning labels from a partially labeled dataset. The first advantage of this approach is that it allows to avoid having to label all the training examples, which is relevant when it is easy to accumulate data but their labeling requires a certain amount of human work. Take for example the classification of images: it is easy to obtain a database containing hundreds of thousands of images, but having for each of them the label that interests us can require a lot of work. Moreover, labels given by humans are likely to reproduce human biases, which a fully supervised algorithm will in turn reproduce. learning semi-supervised can sometimes avoid this pitfall. This is a more advanced topic, which we will not consider not in this book.

  4. Reinforcement learning

    5. In reinforcement learning, the learning system can interact with its environment and perform actions. In return for these actions, he gets a reward, which can be positive if the action was a good choice, or negative if not. The reward can sometimes come after a long series of actions; this is the case for example for a system learning to play go or chess. Thus, learning in this case consists in defining a policy, that is to say a strategy to always obtain the best possible reward. The main applications of reinforcement learning are found in games (chess, go, etc) and robotics. This subject is well beyond the scope of this book.
